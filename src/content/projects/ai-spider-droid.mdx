## Overview

AI Spider Droid is an autonomous multi-legged robot built around the NVIDIA Jetson Nano edge AI platform. It runs PyTorch-based neural networks fully on-device for real-time perception and decision-making, allowing the droid to navigate its environment, avoid obstacles, and respond to visual stimuli without any human input.

## Problem

Most hobby robots rely on hard-coded, rule-based navigation, which breaks down the moment the environment changes. The goal was to prove that real neural network inference can run directly on a small legged robot, keeping the compute on the hardware itself rather than offloading to a remote machine, and producing behavior that adapts instead of following fixed rules.

## My Role

I served as the AI and robotics engineer for the project. I designed the overall system architecture, trained the perception models, built the control loop that turns model outputs into leg movements, and handled the full deployment pipeline onto the Jetson Nano.

## Key Features

- **On-device edge AI inference.** All neural network inference runs in real time on the Jetson Nano itself, with no remote compute or operator.
- **Vision-based obstacle avoidance.** A camera-driven perception pipeline detects obstacles and routes the droid around them.
- **Adaptive gait control.** A Python controller adjusts the leg movement patterns of the servo-driven chassis based on terrain and sensor feedback.

## Tech Stack

- **Compute:** NVIDIA Jetson Nano (4GB) for GPU-accelerated edge inference
- **Perception:** PyTorch CNN for object and obstacle detection, optimized with TensorRT
- **Control:** Python control loop translating perception outputs into servo gait commands
- **Hardware:** Multi-legged spider-style chassis with servo-driven legs, camera module, and IMU for orientation feedback
- **Training:** Models trained offline on a GPU workstation, then exported and deployed to the Jetson

## Development Process

The project moved in three phases. First, the perception models were trained and validated offline on a GPU workstation, where iteration is fast. Second, the trained models were exported and optimized with TensorRT to fit the Jetson Nano's compute and memory budget. Third, the optimized models were integrated with the Python gait controller on the robot, where the perception-to-motion loop was tuned against real-world obstacles and terrain.

## Challenges & Solutions

- **Meeting real-time latency on constrained hardware.** Stock PyTorch models were too slow on the Jetson Nano. Quantizing and optimizing them with TensorRT brought inference within real-time requirements.
- **Power constraints.** Heavy compute drains the battery quickly on a mobile robot. Balancing inference load against battery capacity became a continuous hardware/software co-design exercise.
- **Latency versus accuracy.** Shrinking the model to hit inference speed targets initially cost detection accuracy. Careful architecture choices recovered acceptable accuracy at the required frame rate.

## Final Result

A self-contained legged robot that perceives its surroundings through its own camera, makes decisions with its own onboard neural network, and walks around obstacles on its own. Every part of the pipeline, from pixels to leg motion, runs on the robot itself.

## What I Learned

- Edge AI is a real engineering constraint. Designing for deployment on limited hardware from the start is far easier than trying to optimize after the fact.
- Modern robotics is deeply interdisciplinary. Mechanical design, embedded firmware, and machine learning all had to work in concert, and a weakness in any one layer showed up as a failure of the whole system.
- This project confirmed that the intersection of robotics and machine learning is where I want to keep building.
